#### git命令：

```
1.#安装好git后查看版本号，验证是否安装成功
git --version

2.#设置用户签名，标识区分不同的开发人员（可以随时更改）
git config --global user.name '陈恩来'                    //可以''或者""
git config --global user.email '3197547929@qq.com'       //可以''或者""

3.#初始化一个版本控制，出现隐藏.git文件目录即版本控制成功（进入某个目录后执行，该目录就成为git管理目录）
git init

4.#将工作区的文件提交到暂存区（不执行添加命令，git就不能进行管理，同时执行add命令只是将工作区的文件复制了一份到暂存区）以下是三种方式：
git add 文件名/目录名 （例如：git add 1.txt或者git add file/   --目录文件需要跟上/用来标识它是目录文件）
git add --all  （提交全部文件）
git add .     （提交全部文件，上面的简写）

5.#删除暂存区待提交的文件（移除文件的版本控制，但并不能作为删除本地库（即是删除.git目录来解除项目的版本控制）的命令来使用）
git rm --cached 文件名/目录名

6.#查看文件状态（红色在工作区，绿色在暂存区，其他文件操作记录）
git status

7.#把暂存区的文件提交到本地仓库，形成一条历史版本记录（也叫历史记录）（简单理解：提交到.git目录下）
git commit -m "注释：描述每一次提交的干了什么事" （注释必须写，不写会强制进入一个编辑文件，留下注释才能不报错）

8.#查看历史提交记录
git log（查看最近一次提交及其以前的所有历史版本记录）
git reflog（查看所有历史版本记录，方便查找需要回退的版本号：一个7位16进制数）

9.#版本回退
git reset --hard HEAD^
（--hard：回退时直接回到上个版本添加前的工作区，进行编辑后再重新添加、提交，会强制清除暂存区的所有文件，具有危险性，比如说回退前有文件再暂存区忘记提交，回退后文件消失，无法找回）
（HEAD：回退到当前版本，原地不动；HEAD^/^^/^^^/……：回退到上一/两/三/……个版本或者HEAD~1/2/3/……：回退到上一/两/三/……个版本）
git reset --hard 版本号（提交的记录数前6位及其更多位数）
git reset --soft HEAD^（--soft：[功能一：重写提交记录]回退时直接回到上个版本添加后，提交前的暂存区，可以再提交时重写注释；[功能二：合并记录，减少记录]开发新功能时，发现某个文件需要修改代码，回退到上个版本提交前的暂存区，然后再工作区修改相关文件后，添加到暂存区，最后提交这个文件时，只需要写一个版本注释，减少git log查找出的版本信息记录）
git revert HEAD（回退时与reset不同的是它会新生成一条撤销记录，并且指针master/HEAD指向当前记录，所以此时的命令中的HEAD指的就会是上一个版本，而不是当前版本，所以用revert回退时指针HEAD与reset的HEAD会相差一个版本）

总结：reset回退时指针往前走，指向以前的版本，不会产生历史记录；revert回退时指针往后走，指向新的撤销记录，会产生一个回退记录。同时，revert产生的版本记录，可以让本地库的版本比远程仓库版本高，修改后可以直接往远程仓库推送；远程仓库版本比本地仓库高时，推送会失败，如果用参数-f强制推送会产生冲突

10.#罗列出所有分支（包括主分支master）
git branch或者git branch -a

11.#创建分支
git branch 分支名

12.#切换分支（工作区会自动进入分支所属的工作区）
git checkout 目的分支名

13.#合并分支（首先得先进入主分支，才能执行合并命令）
git merge 要合并的分支名（相当于是主分支指针向前走指向了分支，而不是代码、文件的的合并）
（合并时是自动合并，如果遇到冲突不能合并，需要程序员进入文件手动解决冲突，解决完后将修改的文件重新添加、提交）

14.#合并分支后，分支就没用了，可以删除分支了
git branch -d 分支名（这只能删除本地分支，远程分支依然存在）

15.#连接远程仓库（首先得先创建远程仓库，才能获取到远程仓库地址）
git remote add origin 远程仓库地址
（origin是一个别名，可以任意取。在第一次连接后，以后向相同的远程仓库推送代码时，就可以用别名代替远程仓库地址，不需要再写远程仓库地址）

17.#查看远程仓库地址
git remote -v

18.#删除远程仓库地址
git remote remove origin（删除别名为origin的远程仓库地址）

19.#推送文件到远程仓库（推送成功可以用git branch -a查看远程分支，git branch只能查看本地分支）
git push origin master:master（将本地主分支（左master）推送到远程仓库的主分支（右master），简写为git push origin master,推送主分支以外的分支都是同样的命令，master改为对应的分支名即可）
（第一次推送可以加参数-u,如git push -u origin master，以后就可以简写为git push）

20.#拉取仓库（只拉取更新的且会自动合并到本地仓库）
git pull origin master（拉取远程仓库地址别名为origin的master分支，如果推送的时候加了-u参数，就可以简写为git pull）

21.#拉取别人的远程仓库到自己本地（实质复制别人的仓库一份）
git clone 远程仓库地址（克隆只能克隆主分支，其他分支需要单独克隆git pull origin 其他分支名）

22.#删除远程分支
git push origin :远程分支名（用空的分支去覆盖远程分支，即是删除远程分支）

23.#github/gitee的其他使用
用fork叉下别人的仓库项目到自己的远程仓库，添加自己的代码后向别人发送合并的请求，别人同意后就会合并代码到他的项目中
```
